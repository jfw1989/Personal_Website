Smart Home mit Home Assistant & Node-Red

Ziel war es, mein Zuhause automatisiert, effizient und datenschutzfreundlich zu steuern. Mit Node-Red habe ich individuelle Präsenz- und Lichtsteuerungen umgesetzt, die auf Sensorwerten, Tageszeit und Gewohnheiten reagieren.

Technologien: Home Assistant, Node-Red, JavaScript, MQTT, Thread

Beispiel-Flow (Node-Red): 


// Multiroom Lautsprecher Automation
let zonePlayer = flow.get(msg.topic);
let playingStates = [flow.get("Arbeitszimmer").state, flow.get("Küche").state, flow.get("Schlafzimmer").state, flow.get("Wohnzimmer").state];
let index;
let player;

let messages = [null, null];

if (zonePlayer.state != "playing") {
    if (playingStates.includes("playing")) {
        index = playingStates.indexOf("playing");
        if (index == 0) {
            player = "Arbeitszimmer";
        } else if (index == 1) {
            player = "Küche";
        } else if (index == 2) {
            player = "Schlafzimmer";
        } else if (index == 3) {
            player = "Wohnzimmer";
        }
        messages[0] = { playerName: msg.topic, payload: flow.get(player).coordinator };
    }
}

if (zonePlayer.volume < 16) {
    zonePlayer.volume = 16;
    messages[1] = { playerName: msg.topic, payload: zonePlayer.volume };
} else if (zonePlayer.volume > 40) {
    zonePlayer.volume = 40;
    messages[1] = { playerName: msg.topic, payload: zonePlayer.volume };
}

return messages;

Backup-Server

Konzeption und Einrichtung eines dedizierten Backup-Servers unter Ubuntu zur inkrementellen Sicherung mittels Tanya und rsync. Der Server führt zeitgesteuerte Backups aus und lagert diese verschlüsselt aus.

Technologien: Ubuntu, Tanya, rsync, cron, Bash, SSH

Code-Beispiel (Cronjob):


0 3 * * MON rsync -e "sudo -u username ssh" -aAXv --delete / --exclude={"/dev/*","/proc/*","/sys/*","/tmp/*","/run/*","/mnt/*","/media/*","/lost+found"} backup_server:/mnt/backup-drive
